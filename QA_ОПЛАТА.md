# ✅ QA: оплата, подписки, возвраты, промокоды

Цель — убедиться, что монетизация работает безупречно до запуска на пользователей.
Тестируем в три уровня. Каждый уровень ловит свой класс ошибок.

---

## Уровень 1. Автотесты (запускать всегда, перед каждым деплоем)

```bash
pip install -r requirements-dev.txt
python3 -m pytest -q
```

Покрывают: доступ/лимиты/режимы, гейтинг КБЖУ, выбор модели, расчёт калорий и
макросов, **идемпотентность платежа** (дубль доставки не начисляет дважды),
установку плана, **продление подписки**, пакеты кредитов, промокоды (успех/ошибка),
**возврат с отзывом Premium**, рефералку (бонус обоим, без самоприглашения),
паритет ключей RU/EN. Должно быть «21 passed» (или больше).

> Запускать в CI на каждый push — это основной непрерывный гарант.

## Уровень 2. Тестовая среда Telegram (бесплатные звёзды, реальные платежи)

### Настройка
1. В приложении Telegram войти в тестовую среду (Settings → несколько тапов по
   версии → *Open Test Server*; на десктопе/macOS).
2. В тестовой среде создать бота у @BotFather → получить **тестовый** `BOT_TOKEN`.
3. Запустить бота локально:
   ```bash
   export BOT_TOKEN=<тестовый_токен>
   export OPENAI_API_KEY=sk-...
   export DATABASE_URL=postgresql://...        # можно локальный Postgres
   export TELEGRAM_TEST_ENV=1
   export MONETIZATION_ENABLED=1
   export ADMIN_IDS=<твой_test_user_id>
   export SUPPORT_CONTACT=@you
   RUN_MODE=polling python main.py
   ```
4. Тестовая карта для пополнения звёзд, если попросит: `4242 4242 4242 4242`,
   любой срок/CVC.

### Ручной чек-лист (☐ — отметить после прохождения)

**Лимиты / пэйвол**
- ☐ Исчерпать дневной лимит (3 анализа) → показывается пэйвол «лимит дня».
- ☐ Эмулировать конец бесплатного периода (в БД сдвинуть `created_at` на 31 день
  назад) → пэйвол «период закончился».

**Покупка**
- ☐ Купить **Premium** → приходит подтверждение; в БД `plan='premium'`,
  `premium_until` ≈ +30 дн; в `payments` строка; `/stats` +1; КБЖУ НЕ появился.
- ☐ Купить **Premium+КБЖУ** → `plan='premium_plus'`; в анализах появились Б/Ж/У.
- ☐ Апгрейд: имея premium, купить premium_plus → план сменился, дата продлилась.
- ☐ Купить **пакет** (50 анализов) → `credits += 50`; тратится только после
  дневного лимита.

**Подписка**
- ☐ `/premium` у активного → показывает дату и `/cancelsub`.
- ☐ `/cancelsub` → «автопродление отменено», Premium до конца периода.

**Возврат (админ)**
- ☐ `/refund <user_id> <charge_id>` (charge_id из логов/админки) → возврат прошёл,
  в БД `is_refunded=TRUE`, Premium отозван (если это была подписка).
- ☐ `/refund` от не-админа → отказ.

**Промокоды (создаём в админке → /promos или /addpromo)**
- ☐ `premium_days` код → активация даёт дни Premium.
- ☐ `premium_plus_days` → даёт КБЖУ-тариф.
- ☐ `credits` → начисляет анализы.
- ☐ Повторная активация тем же юзером → отказ «уже активировал».
- ☐ Код после исчерпания лимита активаций → отказ.
- ☐ Истёкший код (дата в прошлом) → отказ.

**Рефералы**
- ☐ Новый юзер по ссылке `?start=ref_<id>` → бонус другу и (по порогу N) рефереру.
- ☐ Повторный `/start` того же юзера → бонус НЕ начисляется снова.
- ☐ Ссылка на самого себя → бонуса нет.

**Монетизация выключена (тумблер в админке)**
- ☐ Всё бесплатно, без лимитов; кнопка Premium скрыта; `/premium` отвечает «бесплатно».

**Языки**
- ☐ Повторить ключевые экраны на EN (Настройки → 🌐 Язык) — тексты переведены.

## Уровень 3. Что нельзя проверить вживую быстро

- **Реальное продление через 30 дней** — ждать месяц не нужно: логика покрыта
  автотестом `test_macros_plan_and_renewal` (эмуляция `is_recurring` + новая дата).

## Перед публичным запуском (прод)
- ☐ Заданы `BOT_TOKEN`, `ADMIN_IDS`, `SUPPORT_CONTACT`; у `zhiromer-admin` — `BOT_TOKEN`, `OPENAI_API_KEY`.
- ☐ `/terms` и `/paysupport` отвечают (требование Telegram).
- ☐ Включена 2FA на аккаунте-владельце бота.
- ☐ Софт-лонч: 5–10 реальных покупок на минимальной цене у узкого круга.
- ☐ Проверен реальный вывод на [Fragment](https://fragment.com): порог и холд.
- ☐ В логах Render видно строку `Оплата: ...` на каждый платёж.
