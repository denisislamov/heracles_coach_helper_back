# 🍏 Жиромер — Telegram-бот для подсчёта калорий

Жиромер (`@zhiromer_bot`) считает калории по фото еды, тексту или числу, ведёт дневной и недельный
отчёты и даёт советы по диете. Распознавание — на **OpenAI GPT-4o (vision)**,
хранение — **PostgreSQL**, деплой — **Render.com** (webhook).

## Возможности

- **Авторизация** — стандартная телеграмовская (по `user_id`), регистрация при `/start`.
- **Цель по калориям** — бот спрашивает её при первом запуске, меняется в меню.
- **Ввод еды тремя способами:**
  - 📷 фото блюда → оценка калорий;
  - 📷 фото + подпись (напр. «овсянка 250 г») → более точная оценка;
  - ✍️ текстовое описание блюда;
  - 🔢 просто число (`350`, `+200`, `350 ккал`) → ручное добавление.
- **Советы по диете** — когда набрано ≥80% цели или есть превышение, бот
  предлагает, что скорректировать (например, заменить сладкое фруктами).
- **Дневной отчёт** — автоматически в заданный час.
- **Недельный отчёт** — раз в неделю в выбранный день.
- **Меню `/menu`** — цель, время дневного отчёта, день недельного, часовой пояс,
  включение/выключение отчётов.
- **Монетизация (Telegram Stars)** — бесплатно N ИИ-анализов в день, далее
  подписка Premium за звёзды (безлимит). Промокоды на дни Premium или кредиты.

## Структура

```
main.py        — точка входа (webhook / polling)
config.py      — переменные окружения
db.py          — PostgreSQL (asyncpg): users, entries, promo_codes, redemptions
ai.py          — GPT-4o: оценка калорий и советы
handlers.py    — команды, приём еды, обработка кнопок
keyboards.py   — инлайн-меню
reports.py     — дневной/недельный отчёт + планировщик (JobQueue)
payments.py    — оплата звёздами, лимиты, подписка, промокоды
feedback.py    — формы обратной связи (баг-репорт, неверные калории)
admin/         — веб-админка (Flask): аналитика, баги, неверные калории
render.yaml    — Blueprint для Render (бот + админка + база)
requirements.txt
.env.example
```

## Обратная связь в боте

Кнопка «🛟 Обратная связь» в меню (или `/feedback`) открывает две формы:

- **🐞 Баг-репорт** — описание текстом + опционально скриншот или видео.
- **🍽 Неверные калории** — что за блюдо и что не так + правильное значение
  (опц.) + фото (опц.).

В обеих формах **ссылки и файлы запрещены** — принимаются только текст и
фото/видео. Всё попадает в админку.

## Веб-админка (`admin/`)

Отдельный Flask-сервис, который ходит в ту же базу PostgreSQL, что и бот.
Деплоится тем же `render.yaml` как сервис `zhiromer-admin`. Вход по логину/паролю.

Три страницы:

1. **📊 Аналитика** — пользователи (всего, новые за 1/7/30 дней), DAU/WAU/MAU и
   stickiness, приёмы пищи, активные Premium, конверсия в оплату, выручка в
   звёздах (≈ в $), открытые баги и замечания; графики за 14 дней.
2. **🐞 Баги** — список баг-репортов с медиа и сменой статуса (new/in_progress/done).
3. **🍽 Неверные калории** — замечания по калорийности; кнопка «✓ В правки»
   сохраняет верное значение в таблицу `food_corrections` (задел на уточнение
   распознавания в будущем).
4. **🎟 Промокоды** — создание кодов (есть пресет «⚡ Бесплатный месяц» = 30 дней
   Premium), список с активациями, включение/выключение. Пользователь активирует
   код в боте командой `/promo КОД`.

**Доступ.** Базовые креды (поменяй после первого входа):

- логин: `admin` · пароль: `Zhiromer-Admin-2026`

В Render → сервис `zhiromer-admin` → Environment задай свои `ADMIN_LOGIN`
(можно `islamov.denis@gmail.com`) и `ADMIN_PASSWORD`. `BOT_TOKEN` нужен админке,
чтобы показывать присланные фото/видео (токен остаётся на сервере, в браузер не
утекает). `SECRET_KEY` Render генерирует сам.

Локальный запуск админки:

```bash
cd admin
pip install -r requirements.txt
DATABASE_URL=postgresql://... BOT_TOKEN=... ADMIN_LOGIN=admin ADMIN_PASSWORD=pw \
  SECRET_KEY=dev python app.py     # http://localhost:8000
```

## Монетизация (Telegram Stars)

> ⚠️ Монетизация **выключена** флагом `MONETIZATION_ENABLED=0` (всё бесплатно,
> без лимитов, кнопка Premium скрыта). Чтобы включить — `MONETIZATION_ENABLED=1`.
> Реализована по стратегии из `МОНЕТИЗАЦИЯ.md`.

Платный только ИИ-анализ (фото/описание) — ручной ввод числа, отчёты, меню и
формы обратной связи бесплатны всегда. Логика доступа (`payments.access_mode`):
свой ключ → безлимит за счёт юзера; иначе Premium → безлимит; иначе кредиты;
иначе `FREE_DAILY_AI` бесплатных в день; дальше — пэйвол.

Что реализовано:

- **Premium-подписка с автопродлением** — нативная подписка Telegram Stars
  (`create_invoice_link(subscription_period=2592000)`), Telegram сам списывает
  каждые 30 дней; `premium_until` двигается по дате от Telegram. Отмена — `/cancelsub`.
- **Пакеты разовых анализов** — `CREDIT_PACKS` (напр. `50:70,200:220`), кнопки в пэйволе.
- **Триал** — `TRIAL_DAYS` дней безлимита новым при первом `/start`.
- **Дешёвая модель для free-тира** — `OPENAI_MODEL_FREE` (`gpt-4o-mini`), платным — `OPENAI_MODEL`.
- **BYOK** — свой ключ OpenAI: `/setkey sk-...` (ключ шифруется Fernet, сообщение
  удаляется), `/delkey`. Включается заданием `BYOK_ENCRYPTION_KEY`.
- **Аудит платежей и возвраты** — каждый платёж пишется в `payments`
  (идемпотентно по `charge_id`); `/refund <user_id> <charge_id>` (админ) делает
  `refund_star_payment` и отзывает Premium.
- **Промокоды** — `/promo КОД`; `/addpromo КОД premium_days|credits ЗНАЧЕНИЕ [макс] [ГГГГ-ММ-ДД]` (админ).
- **Статистика** — `/stats` (админ): юзеры, новые за 7д, активные Premium, платежи и звёзды за 30д.
- **Обязательное по правилам Telegram** — `/terms`, `/paysupport`.

**Вывод денег:** звёзды выводятся в TON через [fragment.com](https://fragment.com)
(после холд-периода), TON продаётся за фиат. Прямой вывод на карту Telegram не
предусмотрен — для цифровых услуг это единственный разрешённый путь.

## Тестирование оплаты (sandbox)

У Telegram есть тестовая среда, где звёзды бесплатные и платежи проходят
понарошку — идеально для проверки покупки Premium без реальных денег.

1. **Создай тестового бота.** В приложении Telegram войди в тестовую среду:
   Settings → пролистай вниз и быстро тапни по версии/иконке настроек несколько
   раз → *Open Test Server* (на Desktop/macOS). Затем в тестовой среде напиши
   `@BotFather` и создай нового бота — получишь **тестовый** `BOT_TOKEN`.
2. **Запусти бота локально в тестовом режиме:**
   ```bash
   export BOT_TOKEN=<тестовый_токен>
   export OPENAI_API_KEY=sk-...
   export DATABASE_URL=postgresql://...   # можно локальный Postgres
   export TELEGRAM_TEST_ENV=1
   export ADMIN_IDS=<твой_user_id>
   RUN_MODE=polling python main.py
   ```
   Флаг `TELEGRAM_TEST_ENV=1` отправляет все запросы на `…/bot<token>/test/…`.
3. В тестовой среде открой своего тестового бота, исчерпай бесплатный лимит и
   нажми «⭐ Premium» — оплата звёздами пройдёт **бесплатно** (тестовая карта
   `4242 4242 4242 4242` для пополнения звёзд, если попросит).
4. Убедись, что после оплаты приходит подтверждение и Premium активируется.

В проде на Render держи `TELEGRAM_TEST_ENV=0` (реальные платежи).

## Локальный запуск

```bash
pip install -r requirements.txt
cp .env.example .env          # заполнить BOT_TOKEN, OPENAI_API_KEY, DATABASE_URL
export $(grep -v '^#' .env | xargs)
RUN_MODE=polling python main.py
```

Нужен локальный или удалённый PostgreSQL. Токен бота — у [@BotFather](https://t.me/BotFather),
ключ OpenAI — на [platform.openai.com](https://platform.openai.com/api-keys).

## Деплой на Render (рекомендуемый путь)

1. Залейте проект в GitHub-репозиторий.
2. На Render: **New → Blueprint**, выберите репозиторий — `render.yaml` создаст
   web-сервис и базу PostgreSQL автоматически.
3. В настройках сервиса задайте секреты (`sync: false`):
   - `BOT_TOKEN`
   - `OPENAI_API_KEY`
4. Deploy. `WEBHOOK_URL` и `DATABASE_URL` подставятся сами; бот сам зарегистрирует
   webhook в Telegram при старте.

### Без Blueprint (вручную)
- Создайте **PostgreSQL** (Free) и скопируйте *Internal Database URL* → `DATABASE_URL`.
- Создайте **Web Service** (Free): Build `pip install -r requirements.txt`,
  Start `python main.py`.
- Env: `BOT_TOKEN`, `OPENAI_API_KEY`, `OPENAI_MODEL=gpt-4o`, `RUN_MODE=webhook`.
  `RENDER_EXTERNAL_URL` Render выставит сам — он используется как адрес webhook.

## Ограничения бесплатного тарифа Render

- Web-сервис **засыпает** после ~15 мин простоя. Первое сообщение после сна
  разбудит сервис, но может прийти с задержкой в несколько секунд.
- Бесплатная база PostgreSQL имеет ограниченный срок жизни — для постоянной
  эксплуатации перейдите на платный план базы.
